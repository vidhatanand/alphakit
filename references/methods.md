# Alphakit Image Output Methods

## Codex Image Generation

For prompt-to-image requests, Alphakit uses Codex image generation as the source-image generator, then applies Alphakit scripts for conversion, transparency extraction, and verification.

Keep these artifacts separate:

- source image generated by Codex image generation
- processed transparent PNG/WebP
- verification report or terminal output

Do not replace generated-image demos with procedural stand-ins. If Codex image generation output is weak, regenerate with a stricter prompt or report the weakness.

## Format Selection

- PNG: Use for transparency, lossless graphics, UI assets, and maximum compatibility.
- JPEG: Use for opaque photographic or illustrative images when smaller file size is desired. JPEG cannot store transparency.
- WebP: Use for web delivery. Use lossless WebP when alpha must be preserved; use quality-controlled lossy WebP for opaque standard images.

Ask for the output format when the user does not specify PNG, JPEG/JPG, or WebP.

# Transparent Image Methods

## Best Method Ranking

1. Codex image generation on a flat chroma-key background, followed by Alphakit background removal and alpha verification.
2. Aligned black/white background extraction.
3. Single solid background removal, only when accepted as approximate.

## Chroma-Key Extraction

Use a flat key color when generating with Codex image generation:

- default key: `#00ff00`
- alternate key: `#ff00ff` for green subjects
- avoid shadows, gradients, floors, reflections, and checkerboards
- avoid the key color inside the subject

Remove the key with:

```bash
python3 scripts/remove_solid_background.py \
  --input source.png \
  --bg "#00ff00" \
  --out transparent.png
```

If the background is not flat enough, regenerate. Do not accept a visibly contaminated edge as a successful transparent asset.

## Black/White Pair Formula

For the same foreground `F` and alpha `a` composited over black and white:

```text
black_rgb = a * F
white_rgb = a * F + (1 - a)
white_rgb - black_rgb = 1 - a
a = 1 - (white_rgb - black_rgb)
F = black_rgb / a
```

This works only when the black and white images are aligned and the foreground is identical.

## Failure Modes

- Two independent generations usually change pose, edge detail, shadows, or object placement.
- Compression, bloom, shadows, and non-solid backgrounds corrupt the formula.
- Single-background removal loses semitransparent edges and struggles with foreground colors close to the background.
- WebP can preserve alpha, but lossy WebP can damage edges. Prefer lossless WebP when WebP is required.
